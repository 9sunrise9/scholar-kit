# 学术写作风格规范

> **规范来源**：基于 WSS-1 写作规范框架  
> **适用范围**：通用学术写作（论文、教材、技术报告）  
> **版本**：1.5  
> **状态**：正式规范  
> **创建日期**：2026-04-18  
> **更新日期**：2026-04-25  

---

## Introduction

本文档定义学术写作的风格规范。适用所有工程技术类学术文本，语言风格须严谨、客观、技术深度适中。

本规范以 PEP 8（Python 代码风格规范）为参照框架，将正文写作的各项要求分为五大类：

* **语言基调**：禁止表达、代词使用、数据引用
* **段落与句子结构**：主题句、判断句、列表引出、标点规范
* **技术内容组织**：介绍顺序、重点标注、交叉引用
* **内容完整性**：实质内容要求、字数目标
* **符号、格式与术语**：标点符号、缩写格式、括号限制

规范采用 **MUST**（必须）、**SHOULD**（应当）、**MAY**（可以）三级约束力标记。MUST 级条款违反即返工，无例外。

## Consistency

本节说明为何要保持一致性，以及何时可以例外。

### A Foolish Consistency is the Hobgoblin of Little Minds

教材的一致性直接影响读者的阅读体验。统一语言风格比个人偏好更重要。

但本规范并非绝对律令。在以下情况下，撰写者可以合理偏离本规范：

1. 应用某条规定会使段落更难以理解时
2. 为与周围已确定风格保持一致时（尤其是引用已有段落时）
3. 有其他充分理由支持偏离时

**然而**：违反本规范铁律条款（MUST 级）不存在上述例外，必须严格遵守。

## Language Tone

本节规定正文语言的基调要求，包括禁止表达、代词使用和数据引用规则。

### Prohibited Expressions

正文 **MUST NOT** 使用以下类型的表达：

| 类别 | 禁止示例 |
|------|---------|
| 感叹式开头 | "在当今科技飞速发展……""随着人工智能浪潮席卷……" |
| 宏大叙事断言 | "引领第四次工业革命""重塑人类文明进程" |
| 文学化比喻 | "赋予设备灵魂""让机器拥有大脑""思考与感知的融合" |
| 能力堆砌句 | "不仅……更……还……"连续出现3个以上层进关系 |
| 演进式隐喻描述 | 用"走向""升级为""蜕变为""扩展到"描述技术范畴或能力变化 |

```good
智能物联网（Artificial Intelligence of Things，AIoT）是将人工智能技术与物联网基础设施深度融合而形成的新型信息技术体系，其核心目标是赋予传统物联网系统自主感知、实时推理与闭环决策的能力。
```

```bad
在当今科技飞速发展的宏大叙事中，AIoT 已成为引领第四次工业革命的核心力量，深刻改变着人类生产生活的方方面面。
```

技术教材应以严谨客观的语言陈述事实，不以修辞代替技术内容。

### Pronoun Usage

指代非人物体、系统、概念、技术时，**MUST** 使用"它"，**MUST NOT** 使用"他"。

```good
智能制造是AIoT最成熟的应用领域，它主要通过在生产线部署传感器网络……实现生产全过程的智能化管控。
```

```bad
智能制造是AIoT最成熟的应用领域，他主要通过在生产线部署传感器网络……实现生产全过程的智能化管控。
```

### Data Citation

凡引用统计数字，**MUST** 在同一句内标注来源机构与年份，格式为：

```
据[机构]统计，[年份]，……[引用标记]
```

```good
据IoT Analytics统计，2023年全球在网物联网设备数量达到163亿台，预计2025年将增至270亿台[1]。
```

```bad
预计到2025年，全球物联网设备数量将突破750亿台，产生的数据量更是无法估量。
```

## Paragraph and Sentence Structure

本节规定段落与句子的结构要求。

### Opening Sentence

每段**第一句**必须是判断句，直接陈述该段核心论点。

**MUST NOT** 以时间状语、条件状语或背景铺垫开头：

```good
边缘计算将数据处理能力从云端向网络边缘迁移，显著降低了端到端通信时延。
```

```bad
随着边缘计算技术的不断成熟……
在工业物联网快速发展的背景下……
```

### Definitive Sentences

描述技术构成、定义、功能关系时，**SHOULD** 优先使用以下句型：

| 句型 | 用途 |
|------|------|
| ……是…… | 定义 |
| ……由……组成 | 构成 |
| ……包括…… | 列举 |
| ……分为…… | 分类 |

**MUST NOT** 连续使用"……可以实现……""……能够……""……支持……"超过2句。

```good
云端模式是指将数据采集设备获取的原始数据全部上传至云端服务器进行集中处理与推理的部署方式。
```

### List Introduction

正文出现4项及以上并列内容时，只有在以下情形下才**SHOULD**使用显式列表：内容构成稳定分类、教学上需要逐项对照、或各项之间需要保持平行结构。若若干内容只是对同一机制的连续补充、工程优化手段的并列说明，或作者为方便组织而临时命名的小标题，**SHOULD** 合并为一段，用"同时""此外""进一步""相应地"等连接词组织。

当确需使用列表时，**MUST** 用以下句式之一引出列表，**MUST NOT** 直接跟列表：

```
……主要包括以下X个方面：
……主要表现在以下几个方面：
……可分为以下X类：
```

```good
AIoT的核心特征主要表现在以下几个方面：
1. 实时推理……
2. 自学习……
3. 分布式协同……
```

```bad
AIoT的核心特征包括：实时推理……、自学习……、分布式协同……
```

### Avoid Ad Hoc Enumerations

对非稳定分类、非标准术语、非教材必须逐项展开的内容，**MUST NOT** 生造枚举标签并逐项独立成段，例如"异步更新：""客户端选择："这类仅为叙述方便而添加的小标题。此类内容**SHOULD**回并到同一段中，通过逻辑连接词说明它们之间的递进、并列或补充关系。

同时，正文**MUST NOT**使用"总述句冒号 + 小标题冒号"的双冒号结构，例如"三种主要优化手段包括：\n\n梯度压缩：……"。应改为单段展开，或改写为"具体而言，……"后直接接续完整句。

```good
通信优化可从梯度传输、聚合时序和参与设备调度三个方面同时推进。其中，梯度压缩通过稀疏化或量化降低单轮上传数据量；异步更新通过取消全局同步等待提升异构设备环境下的训练吞吐量；而客户端选择则通过控制每轮参与设备集合压缩通信时延。
```

```bad
三种主要优化手段包括：

梯度压缩：……

异步更新：……

客户端选择：……
```

### Parallel Punctuation

| 并列成分数量 | 标点规则 |
|-------------|---------|
| 3个及以内 | 用顿号（、）连接 |
| 4个及以上 | **MUST** 用分号（；）分隔 |

```good
采集心电、血氧、血压、血糖等生理参数
感知层；网络层；应用层；平台层
```

### Process Description

描述技术流程、数据处理管道或系统工作过程时，**MUST** 使用完整的学术文字描述，**MUST NOT** 使用箭头符号（ `→` ）串联流程步骤。

**唯一例外**：纯技术参数符号（如量化精度 FP32→INT8）和图表图说（非正文散文）中可保留箭头符号。

```good
AIoT智慧医疗的技术框架分为四个处理层次：可穿戴传感器持续采集生理参数；采集数据经BLE或NB-IoT协议无线上传至网关；边缘节点对敏感数据执行隐私保护计算与本地预处理；处理后的特征数据提交云端AI诊断模型完成疾病分析与早期预警。
```

```bad
AIoT智慧医疗的技术框架为：可穿戴生理传感 → 无线数据传输（BLE/NB-IoT）→ 边缘端隐私保护计算 → 云端AI诊断模型。
```

多阶段流程的推荐句式：

| 阶段数 | 推荐句式 |
|--------|---------|
| 2阶段 | "……首先……；其次……" |
| 3阶段 | "……分为三个阶段：首先……；其次……；最后……" |
| 4阶段及以上 | "……分为X个阶段：……；……；……；……" |
| 层次结构 | "……分为X个层次：……层……；……层……；……" |

## Technical Content Organization

本节规定技术内容的组织顺序与标注方式。

### Standard Sequence

介绍一项技术时，**MUST** 按以下顺序展开，不得颠倒：

```
1. 定义（中英文名称，一句话说明"是什么"）
2. 工作原理（说明"怎么运作"，可含公式或流程）
3. 系统组成（列出关键组件或模块）
4. 分类（如适用：主要类别及其特点）
5. 典型应用（至少一个具体实例，含量化效益指标）
6. 局限性（7.3节及之后 MUST 包含）
```

```good
云端模式是指将数据采集设备获取的原始数据全部上传至云端服务器进行集中处理与推理的部署方式。该模式的主要优点是计算资源充裕、模型训练与更新便捷；主要局限是数据传输时延较高（通常为数百毫秒至秒级），且原始数据离开设备后存在隐私泄露风险，不适用于工业实时控制场景。
```

```bad
云端模式的优势在于计算资源丰富、模型更新灵活，但其致命弱点是高延迟和隐私风险，这在工业实时控制场景中是不可接受的。云端模式是指……（先评价，再补定义）
```

### Emphasis Markers

需强调的内容，**MUST** 使用以下词语引出：

| ✅ 正确 | ❌ 禁止 |
|---------|--------|
| 特别指出，…… | 值得注意的是，…… |
| 需要说明的是，…… | 很重要的是，…… |
| | 不可忽视的是，…… |

### Cross-References

引用本书前序章节内容，**MUST** 用显式句式标明出处：

```good
如第1章所述，感知层是物联网系统最底层的物理设备集合。
本书第2章已介绍的通信协议包括IEEE 802.15.4和LoRaWAN。
```

## Content Completeness

本节规定内容的实质性要求。

### Substantive Content

* 每节**MUST** 包含技术原理 + 至少一个具体应用实例（含量化效益指标）
* **MUST NOT** 只写标题而无实质内容
* 7.3节及之后各小节**MUST** 包含"局限性"部分

### Word Count Target

全书正文目标 **25, 000–30, 000 汉字**（不含代码块、图题、表题、公式符号）。

## Punctuation and Formatting

本节规定标点符号、格式与术语使用的具体要求。

### Chinese Punctuation

除数学公式（ `$...$` 和 `$$...$$` 块）内部以外，**MUST** 使用中文全角标点：

| 符号类型 | ✅ 正确 | ❌ 禁止 |
|---------|--------|--------|
| 引号 | `"……"`  `"'……'"` | `"..."`  `'...'` |
| 括号 | `（……）` | `(...)` |
| 冒号 | ： | : |
| 分号 | ； | ; |
| 逗号 | ， | , |
| 句号 | 。 | . |

### Quotation Mark Usage

正文中的引号**MUST** 使用中文弯引号，**MUST NOT** 使用英文直引号（ `"` ，U+0022）。

**中文弯引号字符**：

| 用途 | 字符 | Unicode |
|------|------|---------|
| 外层开引号 | `"` | U+201C |
| 外层闭引号 | `"` | U+201D |
| 内层开引号（嵌套） | `'` | U+2018 |
| 内层闭引号（嵌套） | `'` | U+2019 |

引号的适用场景仅限以下两种：

1. **术语强调**：首次引入非正式说法或特定用语时，如：形成感知–推理–决策–执行的完整闭环。
2. **直接引语**：引用他人原话或标准文献中的特定表述。

**MUST NOT** 将引号用于以下场景：

* 描述技术范畴或系统能力的升级/演进（见 L4）
* 代替括号说明术语
* 修辞强调（使用引号赋予词语"特殊含义"）

```good
1999年，Kevin Ashton提出物联网概念，强调设备之间的自动化信息交互。
系统决策依赖人工编写的规则，无法自动适应环境变化。
```

```bad
确立了"万物互联"的基本方向。（用引号包裹普通术语）
使物联网系统从"连接工具"升级为"智能系统"。（用引号强化比喻）
```

### Abbreviation Format

英文缩写首次出现，**MUST** 先给全称再给缩写：

```good
联邦学习（Federated Learning，FL）
时间敏感网络（Time-Sensitive Networking，TSN）
```

```bad
联邦学习（FL）
联邦学习FL
```

### English in Parentheses

若某中文术语在本节中不需要缩写，**MUST NOT** 在括号内单独标注该术语的英文翻译：

```good
直接使用：感知层、数据泄露、Q学习
```

```bad
感知层（Perception Layer）
数据泄露（Data Leakage）
Q学习（Q-Learning）
```

**例外**：图表说明、算法名称、产品型号等非散文语境中，可保留英文标识。

### Bracket Usage

正文括号**MUST** 仅限以下两种用途：

1. 注明缩写全称（格式见上）
2. 补充数字参数

**MUST NOT** 用括号对前文的中文术语进行说明或重述：

```good
适应环境分布的变化，即发生概念漂移时也能自动调整。
采集心电、血氧、血压、血糖等生理参数。
```

```bad
适应环境分布的变化（即概念漂移）
可穿戴生理传感（心电、血氧、血压、血糖）
```

### Professional Term Abbreviation Rules

**不常用专业术语缩写必须补全中文全称**。以下缩写在正文中首次出现时，**MUST** 补全中文全称后再给英文全称和缩写：

| 缩写 | 中文全称 | 说明 |
|------|----------|------|
| ARIMA | 自回归积分滑动平均模型 | 时间序列统计方法 |
| Isolation Forest | 隔离森林 | 异常检测算法 |
| One-Class SVM | 单类支持向量机 | 异常检测算法 |
| LSTM-AE | LSTM自编码器 | 自编码器变体 |
| STL | 季节性与趋势分解 | 时间序列分解 |
| TCN | 时序卷积网络 | 时序建模网络 |
| DSP | 数字信号处理器 | 硬件处理器 |
| RTOS | 实时操作系统 | 操作系统类型 |
| REST | 表述性状态转移 | 架构风格 |
| FMCW | 调频连续波 | 雷达感知技术 |
| AGV | 自动导引车 | 工业搬运设备 |
| M2M | 机器对机器 | 通信模式 |
| AE | 自编码器 | 神经网络类型 |
| DBN | 深度信念网络 | 神经网络类型 |
| FedAvg | 联邦平均算法 | 联邦学习聚合算法 |

**缩写后文出现频率决定是否保留括号**：

1. **缩写在正文后续段落中出现≥2次** → 必须保留括号格式：`中文全称（英文全称，缩写）`
2. **缩写在正文后续仅出现1次或从未出现** → 删除括号及缩写，只保留中文全称
3. **例外**：品牌名（NVIDIA、AWS、TensorFlow等）和通用硬件规格（Cortex-A53@1.2GHz）**不适用**上述规则，保持原样

```good
工业物联网（Industrial Internet of Things）——IIoT仅出现1次，删缩写
联邦平均算法（Federated Averaging，FedAvg）——FedAvg在后续多次出现，保留括号
```

```bad
联邦学习（Federated Learning，FL）——FL在后续仅出现1次，删括号
工业物联网（Industrial Internet of Things，IIoT）——IIoT仅出现1次，仍补了缩写
```

**缩写后文出现频率统计方法**：在正文所有段落中（含首次出现段），统计该缩写的独立词匹配次数（正则 `\bABBR\b` ）。

**框架、数据集、硬件型号**：直接使用英文名称，不强制要求中文全称，但首次出现时可括注：

```good
TensorFlow Lite（TFLite）
ImageNet数据集
NCNN（腾讯开源）
Cortex-A53@1.2GHz
```

### Bold Text

正文段落中**MUST NOT** 使用 `**加粗**` 格式：

* 不得以段首加粗文字代替小标题
* 不得用加粗强调句中关键词
* 表题、图题等可正常使用格式

### Dash Separator

正文**MUST NOT** 以 `——` 破折号作为段落或内容之间的分节符。如需区分内容层次，改用分级标题或完整的段落过渡句。

## Formatting Quick Reference

本节提供格式速查，不含新的规范要求。

### Term First Appearance

```
智能物联网（Artificial Intelligence of Things，AIoT）
联邦学习（Federated Learning，FL）
边缘计算（Edge Computing，EC）
知识蒸馏（Knowledge Distillation，KD）
强化学习（Reinforcement Learning，RL）
```

### Figure Numbers

```
图7-1 传统物联网三层架构示意图
图7-2 云–边–端协同架构示意图
```

* 连字符（-），非句点（.）
* 正文先出现"如图7-X所示"，才放置图片

### Table Numbers

```
表7.1 传统 IoT 与 AIoT 特征对比
表7.2 AI 与物联网融合层次对比
```

* 句点（.）
* 表名置于表格**上方**

### Equation Numbers

```
（7.1）：多传感器加权融合公式
（7.2）：真值发现置信度公式
```

* 公式后**必须**跟"式中，…为…；…为…"参数说明

### List Formats

| 场景 | 格式 |
|------|------|
| 3项以内 | 顿号，句中 |
| 4–8项 | `1. 2. 3.` 换行编号 |
| 有子项或长列表 | `-` 引导的列表 |

## Violation Quick Reference

本节汇总常见错误，供快速核查使用。

| 编号 | 类别 | ❌ 错误 | ✅ 正确 |
|------|------|--------|--------|
| L1 | 宏大叙事 | "引领第四次工业革命" | 直接陈述技术事实 |
| L2 | 代词 | "他"指代系统 | "它"指代非人物体 |
| L3 | 数据来源 | "突破百亿台"无引用 | "据IoT Analytics统计，2023年……[1]" |
| L4 | 演进式隐喻 | "从'连接'扩展到'智能赋能'""升级为智能系统""走向万物智联" | 直接描述新增的具体技术功能，如"进一步集成了感知理解、在线学习与自主决策功能" |
| P1 | 段落开头 | "随着……"状语开头 | 判断句直接开段 |
| P2 | 列表引出 | 直接跟列表 | "主要包括以下几个方面：" |
| P2b | 不必要枚举 | 为行文方便生造"异步更新：""客户端选择："等小标题并逐项独立成段 | 合并为单段，用连接词说明并列或递进关系 |
| P2c | 双冒号结构 | "三种主要优化手段包括："后再接"梯度压缩：……" | 改为单段展开，或保留一个引导层级 |
| P3 | 并列标点 | 顿号连用4项以上 | 4项以上用分号 |
| T1 | 技术顺序 | 先讲优劣再补定义 | 定义→原理→构成→分类→应用→局限 |
| T2 | 重点标注 | "值得注意的是" | "特别指出""需要说明的是" |
| F1 | 标点符号 | `"..."`  `"`  `"` | `"……"`  `"'……'"` |
| F1b | 英文直引号 | `"万物互联"` (U+0022) | `"万物互联"` (U+201C/201D) |
| F1c | 引号滥用 | `"连接工具"升级为"智能系统"` | 去掉引号，改用准确技术术语直接陈述 |
| F2 | 缩写格式 | "联邦学习（FL）" | "联邦学习（Federated Learning，FL）" |
| F3 | 括号滥用 | "变化（即概念漂移）" | "变化，即概念漂移时也……" |
| F4 | 加粗 | "**关键**在于……" | 直接陈述，不加粗 |
| F5 | 破折号分节 | "——"分隔段落 | 用标题或过渡句 |
| F6 | 专业缩写缺中文全称 | "ARIMA、Isolation Forest" | "自回归积分滑动平均模型（ARIMA）、隔离森林（Isolation Forest）" |
| F7 | 缩写后文未出现仍补缩写 | "工业物联网（IIoT）"（IIoT全文仅1次） | "工业物联网"（IIoT删缩写） |
| F8 | 框架名/型号直用英文 | "TFLite"、"ImageNet" | 允许直接使用，但首次出现可括注中文（TFLite为TensorFlow Lite） |

### 示例 4：演进式隐喻描述替代技术事实

```bad
AIoT的内涵已经从传统物联网的"连接"扩展到"智能赋能"，使物联网系统从"连接工具"升级为"智能系统"，走向"万物智联"的新阶段。
```

```good
AIoT在传统物联网互联互通基础上进一步集成了感知理解、在线学习与自主决策功能，核心目标在于赋予设备自治性、认知能力与数据驱动的决策能力。
```

**规则**：描述技术范畴或系统能力的变化时，**MUST** 使用具体技术术语（如集成、嵌入、增加了、扩展至某功能等），**MUST NOT** 使用"走向""升级为""蜕变为""扩展到"等带有隐喻或进化含义的词语。

## Appendix C: Changelog

本附录收录原书第1章典型段落，供撰写者参照校准风格。

### 示例 1：定义后接多层次分解

> 对于物联网（IoT）可以给出如下基本定义：物联网是通过信息传感设备，按照约定的协议，把任何物品与互联网连接起来，进行信息交换和通信，以实现智能化识别、定位、跟踪、监控和管理的一种网络。定义中隐含了三层意义：①信息全面感知……②通过网络传输……③智能决策与控制……

### 示例 2：先归纳关键词，再引出编号列表

> 目前对物联网概念的表述，可以将其核心要素归纳为"感知、传输、智能、控制"八个字。物联网的主要特征表现在以下几个方面。
>
> （1）全面感知。利用RFID、传感器、二维码等随时随地获取物体的信息。
>
> （2）可靠传输。通过各种电信网络与互联网的融合，将物体的信息实时准确地传递出去。

### 示例 3：先说方法论依据，再给结论

> 借鉴计算机网络体系结构模型的研究方法，将物联网系统组成部分按照功能分解成若干层次……因此，若从功能角度建构物联网体系结构，可划分为感知层、网络层和应用层3个层级。

### 示例 1：感叹式开头、宏大叙事

在当今科技飞速发展的宏大叙事中，智能物联网（AIoT）……已成为引领第四次工业革命的核心力量，深刻改变着人类生产生活的方方面面。
