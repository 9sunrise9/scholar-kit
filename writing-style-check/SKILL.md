# 学术写作风格检查

本技能依据通用学术写作规范提供风格检查与指导。

## 核心原则

**MUST（必须）**：违反即返工，无例外
**SHOULD（应当）**：强烈建议遵守
**MAY（可以）**：可选

## 语言基调

### 禁止表达

| 类别 | 禁止示例 |
|------|---------|
| 感叹式开头 | "在当今科技飞速发展……""随着人工智能浪潮席卷……" |
| 宏大叙事断言 | "引领第四次工业革命""重塑人类文明进程" |
| 文学化比喻 | "赋予设备灵魂""让机器拥有大脑" |
| 能力堆砌句 | "不仅……更……还……"连续3项以上层进 |
| 演进式隐喻 | 用"走向""升级为""蜕变为"描述技术变化 |

### 代词使用

指代非人物体、系统、概念时，**必须**用"它"，**禁止**用"他"。

### 数据引用

凡引用统计数字，**必须**在同一句内标注来源机构与年份：
```
据[机构]统计，[年份]，……[引用标记]
```

## 段落与句子结构

### 段落开头

段落第一句**必须**是判断句，直接陈述核心论点。**禁止**以时间状语、条件状语或背景铺垫开头。

### 列表引出

4项及以上并列内容**必须**用以下句式引出，**禁止**直接跟列表：
```
……主要包括以下X个方面：
……主要表现在以下几个方面：
……可分为以下X类：
```

### 标点规则

| 并列成分数量 | 标点规则 |
|-------------|---------|
| 3个及以内 | 顿号（、） |
| 4个及以上 | **必须**用分号（；） |

### 流程描述

**禁止**使用箭头符号（`→`）串联流程步骤。**必须**使用完整的学术文字描述。

### 技术介绍顺序

介绍一项技术时，**建议**按以下顺序展开：
1. 定义（中英文名称）
2. 工作原理
3. 系统组成
4. 分类（如适用）
5. 典型应用（含量化效益指标）
6. 局限性

### 重点标注

| ✅ 正确 | ❌ 禁止 |
|---------|--------|
| 特别指出，…… | 值得注意的是，…… |
| 需要说明的是，…… | 很重要的是，…… |

## 标点与格式

### 中文标点

除数学公式内部外，**必须**使用中文全角标点。

### 引号

**必须**使用中文弯引号（`"……"` `"'……'""`），**禁止**使用英文直引号。

### 缩写格式

首次出现**必须**先给中文全称再给英文全称和缩写：
```
联邦学习（Federated Learning，FL）
```

### 括号限制

正文括号**仅限**两种用途：
1. 注明缩写全称
2. 补充数字参数

**禁止**用括号对前文术语进行说明或重述。

## 快速核查

常见违规速查：

| 编号 | 类别 | 错误 |
|------|------|------|
| L1 | 宏大叙事 | "引领第四次工业革命" |
| L2 | 代词 | "他"指代系统 |
| L3 | 数据来源 | "突破百亿台"无引用 |
| P1 | 段落开头 | "随着……"状语开头 |
| F1 | 标点符号 | 英文直引号 |
| F2 | 缩写格式 | "联邦学习（FL）" |
| F5 | 破折号分节 | "——"分隔段落 |

## 自动检查脚本

运行写作风格检查：

```bash
python3 scripts/writingstylecheck.py <文件路径>
python3 scripts/writingstylecheck.py --text "你的文本"
python3 scripts/writingstylecheck.py --demo  # 运行演示
```

脚本可检测：
- L节（语言基调）：感叹式开头、宏大叙事、文学化比喻、代词使用、数据引用
- P节（段落结构）：段落开头、列表引出、并列标点
- T节（技术内容）：重点标注词语
- F节（符号格式）：引号、括号、缩写格式、加粗、破折号分节
